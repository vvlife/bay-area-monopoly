## ❌ 问题清单

### 问题 #1: 联机模式UI流程问题

**严重程度**: 高
**状态**: 🔴 需要修复

**问题描述**:
玩家1在测试中进入了单人模式而不是联机模式。测试结果显示"进入游戏"按钮可见，这表明`selectMode('multi')`可能没有正确执行。

**截图分析**:
- 玩家1截图显示单人模式UI（"进入游戏"按钮可见）
- 玩家2截图显示需要输入房间号

**根本原因**:
- 联机模式的房间创建流程与单人模式不同
- 玩家1点击开始后，联机模式会显示房间号而不是直接进入游戏
- 测试脚本的预期行为与实际UI流程不匹配

**修复建议**:
1. 修改测试脚本，在联机模式下正确获取房间号
2. 或者在UI中添加一个"创建房间"按钮的显式选择

### 问题 #2: Socket.io 对象未初始化

**严重程度**: 中
**状态**: 🔴 需要修复

**错误信息**:
```
TypeError: Cannot read properties of null (reading 'emit')
TypeError: Cannot read properties of null (reading 'disconnect')
```

**问题描述**:
`page.evaluate()` 中访问 `socket` 对象时返回 `null`。这是因为 `socket` 在页面加载后通过 `io()` 初始化，但测试执行时机可能过早。

**修复建议**:
```javascript
// 检查socket是否已初始化
await this.page.evaluate(() => {
  if (typeof socket !== 'undefined' && socket) {
    socket.emit('joinRoom', { roomId: roomId, name: '玩家2' });
  }
});
```
